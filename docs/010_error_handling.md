# 010 エラーハンドリング

## 概要
各機能でのエラー処理と適切なユーザー通知実装

## 要件
- ネットワークエラー処理
- API呼び出しエラー処理
- ファイル処理エラー処理
- 権限エラー処理
- 非破壊的なリトライ機能

## 完了条件
- 各種エラーが適切に捕捉される
- ユーザーに分かりやすいエラーメッセージが表示される
- リトライ可能なエラーでリトライ選択肢が提示される
- エラー発生時に画像が端末に残らない
- 致命的エラーでもアプリがクラッシュしない

## エラー種別
- **ネットワークエラー**: 「通信に失敗しました。再試行しますか？」
- **API処理エラー**: 「画像処理に失敗しました。再試行しますか？」
- **大容量ファイル**: 「ファイルサイズが大きすぎます。」
- **権限エラー**: 「権限が必要です。設定で許可してください。」
- **一般エラー**: 「エラーが発生しました。やり直してください。」
